HDFS
    - Hadoop Distributed File System
    - 대용량의 파일을 분산된 서버에 저장하고, 다양한 클라이언트가 저장된 데이터를 처리할 수 있도록 설계된 파일 시스템이다.
    - 저사양 서버를 이용해서 대용량의 스토리지를 구성하고, 하나의 스토리지처럼 사용할 수 있다.
    - HDFS가 제공하는 API를 이용하면 파일을 읽고 쓸 수 있다.
    
    - HDFS의 목표
        - 장애복구
            - 장애를 감지하고, 대처할 수 있다.
            - 데이터를 저장하면 복제본을 함꼐 저장해서 데이터 유실을 방지한다.
        - 스트리밍 방식의 데이터 접근
            - 클라이언트는 연속된 흐름으로 데이터에 접근할 수 있다.
        - 대용량 데이터 저장
            - 대용량의 데이터를 저장할 수 있다.
        - 데이터 무결성
            - HDFS는 한 번 저장한 데이터는 수정할 수 없고, 읽기만 가능하다.
            
    - HDFS 아키텍처
        - 블록구조 파일시스템
            - HDFS는 저장하는 파일을 특정 크기의 블록으로 나누어서 분산된 서버에 저장한다.
            - 기본 블록크기는 128MB다.
            - HDFS는 블록을 저장할 때 복제본을 함께 저장한다.(기본 복제크기는 3개다
            
        - 네임노드와 데이터노드
            - HDFS는 마스터-슬레이브 아키텍처다.
            - 마스터는 네임노드, 슬레이브는 데이터노드다.
            - 네임노드
                - 메타데이터 관리
                    - 파일시스템을 유지하기 위한 메타데이터(파일명, 디렉토리, 크기, 권한, 블록매핑정보)를 관리한다.
                    - fsimage와 editslog 파일을 사용해서 메타데이터를 관리한다.
                - 데이터노드 모니터링
                    - 데이터노드는 네임노드로 하트비트(데이터노드 상태정보, 블록의 목록정보)를 주기적으로 전송한다.
                    - 네임노드는 하트비트를 전송하지 않는 경우 장애가 발생한 노드로 간주한다.
                - 블록관리
                    - 장애가 발생한 데이터노드의 블록을 새로운 데이터노드로 복제한다.
                    - 용량이 부족한 데이터노드가 있으면 여유가 있는 데이터노드로 블록을 이동시킨다.
                - 클라이언트의 요청 접수
            - 데이터노드
                - 파일을 로컬디스크에 저장한다.
            - 보조 네임노드  
                - 보조 네임노드는 주기적으로 네임노드의 fsimage를 갱신한다.
                
        - HDFS shell command
            - 파일 목록 보기
                hadoop fs -ls <디렉토리명>
                hadoop fs -ls -r <디렉토리명>
            - 파일의 내용 보기
                hadoop fs -cat <파일명>    * 텍스트 파일만 출력한다.
                haddop fs -text <파일명>   * 테스트 파일을 출력한다. 압축된 파일도 텍스트형태로 출력한다.
            - 디렉토리 생성
                hadoop fs -mkdir <디렉토리명>
                hadoop fs -mkdir -p <디렉토리명/디렉토리명>
            - 파일 복사
                - 로컬 파일시스템의 디렉토리나 파일을 HDFS로 복사
                    hadoop fs -put <로컬디렉토리|로컬파일> <목적지디렉토리|목적지파일>
                    hadoop fs -copyFromLocal <로컬디렉토리|로컬파일> <목적지디렉토리|목적지파일>
                - HDFS에 저장된 데이터를 로컬 파일시스템으로 복사
                    hadoop fs -get <소스디렉토리|소스파일> <로컬디렉토리|로컬파일>
                    hadoop fs -copyToLocal <소스디렉토리|소스파일> <로컬디렉토리|로컬파일>
                    hadoop fs -getmerge <소스디렉토리|소스파일> <로컬디렉토리|로컬파일>
            - 파일 디렉토리 삭제
                hadoop fs -rm <디렉토리|파일>
                hadoop fs -rm -r <디렉토리|파일> * 비어있지 않는 디렉토리도 삭제한다.
            - 파일의 마지막 내용 확인
                hadoop fs -tail <파일>
                
        - HDFS API
            - FileSystem
                - 파일시스템과 인터페이스하기 위한 추상 클래스다.
                - HDFS와 로컬 파일 시스템을 처리하기 위한 하위 클래스를 가지고 있다.
                - HDFS 파일시스템과 인터페이스하는 파일시스템 객체의 생성 
                    FileSystem hdfs = FileSystem.get(new Configuration());
                - 로컬 파일시스템과 인터페이스하는 파일시스템 객체의 생성
                    FileSystem local = FileSystem.getLocal(new Configuration());
            - Path
                - 파일과 디렉토리를 표현하는 객체다.
            - FileStatus
                - 파일과 디렉토리의 메타데이터를 저장하는 객체다.
                - 파일의 크기, 접근권한, 수정날짜와 등과 같은 메타데이터를 표현한다.
            - FSDataInputStream과 FSDataOutputStream
                - DataInputStream과 DataOutputStream의 하위클래스다.
                - 랜덤엑세스 기능을 가지고 있다.
