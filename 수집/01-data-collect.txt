데이터 수집의 정의
	- 서비스 활용에 필요한 데이터를 시스템의 내부 혹은 외부에서 주기성을 갖고 필요한 형태로 수집하는 할동
	

수집 데이터의 분류
	- 데이터의 형태에 따른 분류
		- 정형 데이터
			관계형데이터베이스의 테이블과 같이 고정된 구조를 가지는 데이터와 스프레드시트 파일
			데이터의 스키마를 지원함
			정형 데이터의 탐색 과정은 테이블 탐색, 컬럼구조 탐색, 로우 탐색 순으로 정형화 되어 있음

		- 반정형 데이터	
			데이터 내부에 정형데이터의 스키마에 해당하는 메타데이터를 가지고 있음
			일반적으로 파일형태로 존재함
			데이터 내부에 있는 규칙성을 파악해 데이터를 파싱할 수 있음
			html, xml, json, 웹로그, 센서 데이터

		- 비정형 데이터
			언어 분석이 가능한 텍스트 데이터나 이미지, 동영상 같은 멀티미디어 데이터가 대표적인 비정형 데이터다.
			웹에 존재하는 소셜미디어 데이터도 비정형 데이터로 구분할 수 있다.
			이진 파일 형태의 데이터일 때는 데이터의 종류에 맞는 응용소프트웨어를 이용해야 한다.

	- 데이터 수집 난이도
		정형 데이터 < 반정형 데이터 < 비정형 데이터
	- 데이터의 잠재적 가치
		저형 데이터 < 반정형 데이터 < 비정형 데이터


데이터 수집절차
	- 데이터 수집 프로젝트의 특성
		- 데이터 입력 측면
			수집 프로세스 자체가 데이터 발생의 주체가 된다.
			수집 정책 관련 기술에 대한 데이터 오너십이 존재한다.
			데이터는 프로세스 하나에서 여러 건 발생할 수 있다.

		- 데이터 처리 측면
			대부분 데이터에 대한 전처리 과정이 필요하다.
			수집한 데이터에 대한 후처리(다른 데이터와의 병합)가 이뤄진다.

		- 서비스 제공 측면
			처리된 데이터를 분리,병합해 새로운 인사이트를 제공하는 서비스가 주를 이룬다.
			재생산의 과정을 거쳐 생성된 데이터가 제공된다.
			데이터 분석 관련 프로젝트가 주를 이룬다.
	
	- 데이터 수집절차 설계
		- 데이터 선정
			수집 가능성, 보안문제, 데이터 정확성, 수집 난이도, 수집 비용 등을 고려해서 수집 데이터를 선정한다.

		- 수집 세부계획 수립
			수집데이터의 위치 파악, 데이터의 유형 파악, 수집시 적용할 기술 및 보안사항 등을 점검한 후 수집 계획서를 작성한다.
			수집 계획서에는 데이터 소스, 수집주기, 수집방법등을 구체화해야 한다.

		- 테스트 수집 진행
			수집 계획서가 완성되면 테스트 수집을 진행한다.
			테스트 수집 후 수집 테스트에 대한 기술적, 업무적 검토를 실시한다.
		

데이터 수집 기술
	- HTTP 수집
		- 크롤링 기술, Open API 수집 기술
		- 웹에서 텍스트 정보를 가져오는 크롤링 수집 기술과 웹을 운영하는 운영주체가 정보를 제공하는 Open API 수집기술
	- 로그/센서 수집
		- 로그 수집 기술, 센서 데이터 수집 기술
		- 데이터 처리 에이전트를 이용해서 로그 및 실시간 센서 데이터 수집기술.
	- DBMS 수집
		- DB에 직접 연결해서 데이터 수집
	- FTP 수집
		- 대용량 파일을 수집하기 위한 클라이언트 서버 간 연결 및 파일전송, FTP 보안 기능이 제공된다.
